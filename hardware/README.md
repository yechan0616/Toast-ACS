# 하드웨어 — 게이트

개찰구는 2개예요. 하나는 입장 전용, 하나는 퇴장 전용이에요. 각 개찰구에 초음파 센서, OLED, 서보가 하나씩 붙어요.

방향은 서버가 확인해요. 입장 버튼은 입장 개찰구 앞에서만, 퇴장 버튼은 퇴장 개찰구 앞에서만 통해요. 반대편에서 누르면 거부돼요.

보드는 2장이에요. Nano ESP32가 서버 통신과 센서, OLED, 판단을 맡고, Uno R3가 서보를 움직여요. 배선은 `pin_map.md`에 정리돼 있어요.

```
기본:  서버 ← 핫스팟 → ESP32 ← UART → Uno R3
백업:  서버 ← 맥북(bridge/) ← USB → ESP32 ← UART → Uno R3
```

서버 연결은 `config.h`의 `USE_WIFI`로 골라요.

- Wi-Fi 모드(`USE_WIFI 1`, 기본): ESP32가 폰 핫스팟으로 서버에 직접 붙어요. 보조배터리만 있으면 컴퓨터 없이 동작해요.
- USB 모드(`USE_WIFI 0`, 백업): ESP32를 맥북에 꽂고 브리지를 실행해요. 전파 환경과 무관해서 핫스팟이 안 붙을 때 쓰기 좋아요.

Nano ESP32는 클래식 블루투스가 안 되고 BLE만 돼요. 무선이 필요하면 Wi-Fi 모드를 쓰세요.

## 부품

| 부품 | 수량 | 쓰임 |
|---|---|---|
| Arduino Nano ESP32 | 1 | 서버 통신, 센서, OLED, 판단 |
| Arduino Uno R3 | 1 | 서보 구동 |
| HC-SR04 초음파 센서 | 2 | 개찰구 앞 사람 확인 |
| OLED SSD1306 128×64 (I2C) | 2 | 좌석 지도, 승인·거부 표시 |
| SG90 서보 | 2 | 차단바 (0°=닫힘, 90°=열림) |
| 저항 1kΩ, 2kΩ | 각 2 | 초음파 Echo 분압 |
| NeoPixel LED 16개 | 옵션 | 모형 좌석 표시. 없으면 OLED 좌석 지도로 충분해요 |

## 동작

- 서버가 방향별로 개방을 내려요. `openIn`이면 입장 쪽만, `openOut`이면 퇴장 쪽만 3초 열려요.
- 닫히는 순간에 초음파가 사람을 감지하면 최대 7초까지 닫힘을 미뤄요. 문 끼임을 막기 위해서예요.
- 입장 OLED는 평소에 좌석 현황 지도를 보여줘요. 입장하면 "입장 승인 / 좌석 A3"가 떠요. 퇴장 OLED는 "통과 승인 / 안녕히 가세요"를 띄워요.
- 공격이 차단되면 OLED에 "공격 차단"과 사유가 2.5초 떠요. 입장 관련은 입장 쪽, 퇴장 관련은 퇴장 쪽, 토큰 재사용이나 위조 티켓 같은 공통 보안은 양쪽에 떠요.
- ESP32와 R3 사이 링크가 1.5초 끊기면 R3가 알아서 문을 닫아요.
- 서보가 반대로 움직이면 `uno-actuator.ino`의 `SERVO_OPEN_DEG`, `SERVO_CLOSE_DEG`를 바꾸면 돼요.

## 펌웨어 올리기

1. Arduino IDE 보드 매니저에서 "Arduino ESP32 Boards"를 설치하고, 보드를 "Arduino Nano ESP32"로 선택해요. 라이브러리는 ArduinoJson, U8g2, Adafruit NeoPixel 3개를 설치해요.
2. `config.h.example`을 `config.h`로 복사하고 설정해요.
3. `gate-firmware/gate-firmware.ino`를 올려요.
4. 보드를 "Arduino Uno"로 바꾸고 `uno-actuator/uno-actuator.ino`를 올려요. 추가 라이브러리는 없어요.

보드 선택을 헷갈리기 쉬워요. gate-firmware는 Nano ESP32, uno-actuator는 Arduino Uno예요.

## Wi-Fi 모드

1. `config.h`에서 `USE_WIFI 1`로 바꾸고 핫스팟 이름과 비밀번호, 서버 주소(`API_BASE`), 게이트 키를 넣어요. 폴백 핫스팟도 넣어두면 자동으로 갈아타요.
2. ESP32에 보조배터리를 꽂으면 끝이에요. 부팅하면 알아서 서버에 붙어요.
3. admin 대시보드에서 게이트가 온라인으로 뜨는지 확인하세요. 원격 개방 버튼으로 서보까지 움직이면 정상이에요.

서버 주소는 맥북 로컬(`http://맥IP:8080`)이나 Cloudflare Tunnel 주소를 넣으면 돼요.

## USB 모드

1. 맥북에서 백엔드를 켜고 ESP32를 USB로 연결해요.
2. 브리지를 실행해요.

   ```sh
   cd hardware/bridge
   npm install
   npm start
   ```

3. 콘솔에 점이 0.7초마다 찍히면 정상이에요. 개방 명령이 지나가면 `→ 게이트 명령 전달: OPEN`이 떠요.

브리지와 Arduino IDE 시리얼 모니터는 같은 포트를 함께 못 써요. 펌웨어를 올릴 때는 브리지를 잠깐 꺼두세요.

## 남은 일

- Wi-Fi 모드가 서버 인증서 검증을 건너뛰고 있어요. 운영 전에 인증서 핀닝으로 바꿔야 해요.
- 통과 인원 카운트(`passedCount`)는 아직 0만 보내요. 센서를 붙이면 서버의 테일게이팅 경보가 바로 동작해요.
- 문 열림 방치 감지는 리드 스위치를 붙이고 폴링에 `doorOpen` 필드를 추가하면 돼요.
